# Courses プロジェクト

## 概要
GitHub Pages で公開するプログラミング講座サイト。

## デザイン方針
- 各言語は独立した React SPA（単一HTML）
- 紫系グラデーション基調、Noto Sans JP + JetBrains Mono
- mojo/index.html がデザインのリファレンス

## ファイル構成
- index.html: ランディングページ（言語一覧）
- {言語名}/index.html: 各講座ページ
- 各ファイルは依存なしの単一HTMLで完結させる

## 講座の構成パターン
- 常設ヘッダー（ロゴがホーム `../index.html` / `index.html` に戻るリンク）
- ヒーローセクション（タイトル、章数、演習数、**コース全体のゴールを示す説明文**）
- 章ごとのアコーディオン
  - 章ヘッダー直下に **🎯 この章のゴール** ブロック（`chapter.goal` フィールド、紫系の枠で強調）
  - 各セクションは `title` + **`body`（必ず文章での解説を 2〜3 段落）** + 任意の `table` / `code` / `afterTable` / `afterCode`
  - コードやテーブルの直後には **`afterCode` / `afterTable` で「ポイント」「ミニ演習」「注意」等のフォローアップ文章を添える**
- 演習問題（難易度⭐表示、コード + 回答の開閉）
- 前後章ナビゲーション（章を切り替えた際は全アコーディオンを閉じる: `Section` / `Exercise` コンポーネントの key に `ch.id` を含めて強制 remount）
- コース最終章の最後、または完了画面の「コース完了」バッジはホーム (`../index.html`) へのリンクにする

## 執筆ルール（セクション単位）
- `body` は箇条書きのみで済ませず、**必ず「なぜそうなっているのか」「Python と比べてどう違うのか」「実務でどう使い分けるのか」を文章で説明する**
- コードだけ・テーブルだけの無言セクションを作らない。必ず前後に文章を置く
- `chapter.goal` は「この章を終えた時点で何ができるようになっているか」を具体的に書く
- コース冒頭（ヒーロー）にはコース全体の学習ゴールを 1 段落で記載する

## 各講座の情報量
- Python既習者を対象とする。
- 入門/初級/中級/上級 ぐらいのレベル分けを設ける。履修完了後の理解度の目安は以下。これを実現するため、それぞれのレベルは大体50時間以上の内容・情報量を必ず設ける。
  - 入門: その言語の基礎や最低限把握しておくべき情報を学ぶ。
  - 初級: その言語を使って、基礎的なアプリを作ることができる。章全体を通して、1つのまとまったアプリの実装などを進め、その過程で学ぶべき新たな要素を1つずつ習得していく形にしたい。
  - 中級: ある程度調べながらであれば、実務でも戦っていけるレベルになる。
  - 上級: その言語の情報を大体つかみ、実務で使ったり、ライブラリを構成したりできるようになる。

## コンテンツ品質ルール
- 教材作成前に、必ず公式ドキュメント（最新リファレンス）を確認し、内容を理解してから書くこと。
- 全ての教材に、その言語のどのバージョン時点の情報かを明記すること（フッター等に記載）。
